# AGENTS.md

## 项目标识

- 项目名称：`MirageTCP`
- 仓库风格名称：`mirage-tcp`
- 当前目标：实现一个本地 TCP 终止库，接收被拦截的 `IPv4/TCP packet` 输入，在本地模拟远端对端，并通过回调将伪造的下行数据包回传给宿主。

## 范围边界

- `MirageTCP` 本身不负责抓包或注入数据包。
- 数据包拦截与重新注入由宿主应用负责，例如通过 `WinTun`。
- `MirageTCP` 负责：
  - 解析入站 `IPv4/TCP packet`
  - 基于五元组跟踪流
  - 模拟被动侧 TCP 握手和最小连接生命周期
  - 生成下行伪造 `IPv4/TCP packet`
  - 通过回调暴露连接生命周期与负载事件

## 当前公共 API 预期

- 输入数据包的 API 使用 `const void*` 加 `size_t`；不要改成 `std::vector<uint8_t>`。
- 主入口类型是 `mirage_tcp::MirageTcp`。
- 当前对外有意义的操作包括：
  - `handle_incoming_ip_packet(...)`
  - `send_downstream_tcp_payload(...)`
  - `close_flow(...)`
- 流通过 `FiveTuple` 标识。

## 命名规则

- 在面向用户的文本和文档中优先使用 `MirageTCP`。
- 在仓库/分发风格标识中优先使用 `mirage-tcp`。
- 除非用户明确要求重命名，否则保持现有 C++ namespace 和 include 路径稳定。

## 实现规则

- 保持设计由宿主驱动且具备确定性。
- 避免在库内部引入后台线程、隐藏定时器或 socket 依赖。
- 将数据包解析与数据包生成同流/状态管理分离。
- 修改应保持最小且局部化；没有必要时不要重写已经工作的协议路径。
- 优先使用简单状态迁移，而不是为了“完整性”做推测式实现。
- 当前阶段，应优先保证本地终止路径的正确性，而不是广泛覆盖 TCP 特性。

## 协议约束

- 聚焦于本地终止的 `IPv4/TCP`。
- 对未支持或部分支持的内容，应保持明确文档说明。
- 处理无效数据包或非法流状态时，优先选择显式 reset/cleanup，而不是静默接受。
- 一旦已经发送服务端侧 payload 或 `FIN`，序列号与确认号处理必须保持严格。

## 测试预期

- 每次协议行为变更都要新增或更新测试。
- 优先使用端到端的 packet 级测试，而不是过度 mock 的单元测试。
- 至少覆盖：
  - 握手成功
  - payload 接收与确认
  - 下行 payload 注入
  - 本地主动关闭
  - 无效流 / 无效 acknowledgment 的 reset 行为
